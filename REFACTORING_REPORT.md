### 2. Апгрейд REST API (`app/api/ide/route.ts`)
**Статус:** ✅ Завершено

**Описание:** Расширение IDE Gateway API новыми действиями для работы с задачами в "Автопилоте".

#### Добавленные функции:

##### GET методы:

**`action === 'get_next_task'`**
- **Параметр:** `projectId`
- **Логика:**
  - Ищет задачи проекта со статусом `TODO`
  - Критически важно: фильтрует задачи, у которых **ВСЕ** блокирующие зависимости имеют статус `DONE`
  - Сортирует по дате создания (`createdAt asc`)
  - Возвращает первую подходящую задачу
- **Возврат:** `{ task: {...} }` или `null` если задач нет

##### POST методы:

**`action === 'generate_prompt'`**
- **Параметры:** `taskId`
- **Логика:**
  - Вызывает `generateTaskPrompt(taskId)` из `lib/agents/prompt-generator`
  - Результат автоматически сохраняется в БД
- **Возврат:** `{ success: true, prompt: "..." }`

**`action === 'add_comment'`**
- **Параметры:** `taskId`, `content`, `role` (опционально, дефолт "DEVOPS")
- **Логика:** Создаёт запись в таблице `Comment`
- **Возврат:** `{ comment: {...} }`

#### Сохранённые функции (для совместимости):

- `action === 'list_tasks'` - список всех задач проекта
- `action === 'read_task'` - чтение задачи по ID
- `action === 'update_task_status'` - обновление статуса задачи

**Результат:** REST API готов для интеграции с IDE и "Автопилотом"

### 3. Настройка "Мозга Автопилота" (.cursorrules)
**Статус:** ✅ Завершено

**Описание:** Обновление файла `.cursorrules` для интеграции с Cursor IDE.

#### Новые команды:

| Команда | Описание |
|---------|----------|
| `orchestrator help` | Справка по доступным командам |
| `orchestrator task` | Получить следующую доступную задачу (через `get_next_task`) |
| `orchestrator prompt <ID>` | Сгенерировать промпт для задачи |
| `orchestrator report <ID> <TEXT>` | Отправить отчёт о выполнении (создаёт комментарий) |
| `orchestrator done <ID>` | Закрыть задачу со статусом DONE |
| `autopilot` | 🚀 Запуск автоматического цикла выполнения всех задач |

#### Алгоритм работы `autopilot`:

1. **Получить задачу:** `@orchestrator task`
2. **Проверка:** Если задач нет (null) → СТОП, вывод "Все задачи выполнены!"
3. **Обработка задачи:**
   - a. Проверить `generatedPrompt`. Если пусто → `@orchestrator prompt <ID>`
   - b. **ВЫПОЛНЕНИЕ:** Прочитать промпт и реализовать требования (код, файлы)
   - c. **ОТЧЁТ:** Собрать список файлов + итог → `@orchestrator report <ID> "Выполнил: ..."`
   - d. **ФИНАЛ:** Закрыть задачу → `@orchestrator done <ID>`
   - e. **РЕКУРСИЯ:** Вернуться к шагу 1

**Результат:** Cursor IDE может работать в полностью автоматическом режиме

## API Endpoints

### GET /api/ide

#### Получить следующую задачу:
```bash
curl "http://127.0.0.1:3002/api/ide?action=get_next_task&projectId=<PROJECT_ID>"
```

**Ответ:**
```json
{
  "task": {
    "id": "...",
    "title": "...",
    "description": "...",
    "status": "TODO",
    "generatedPrompt": "...",
    "dependencies": [...]
  }
}
```

### POST /api/ide

#### Сгенерировать промпт:
```bash
curl -X POST "http://127.0.0.1:3002/api/ide" \
  -H "Content-Type: application/json" \
  -d '{"action":"generate_prompt", "taskId":"<TASK_ID>"}'
```

**Ответ:**
```json
{
  "success": true,
  "prompt": "Сгенерированный промпт..."
}
```

#### Добавить комментарий (отчёт):
```bash
curl -X POST "http://127.0.0.1:3002/api/ide" \
  -H "Content-Type: application/json" \
  -d '{
    "action":"add_comment",
    "taskId":"<TASK_ID>",
    "content":"Выполнил: создал файлы X, Y, Z",
    "role":"CURSOR"
  }'
```

**Ответ:**
```json
{
  "comment": {
    "id": "...",
    "taskId": "...",
    "content": "...",
    "authorRole": "CURSOR",
    "createdAt": "..."
  }
}
```

#### Обновить статус задачи:
```bash
curl -X POST "http://127.0.0.1:3002/api/ide" \
  -H "Content-Type: application/json" \
  -d '{"action":"update_task_status", "taskId":"<TASK_ID>", "status":"DONE"}'
```

## Технические детали

### Зависимости задач

Схема Prisma использует самореференсную связь `TaskDependencies`:

```prisma
model Task {
  id              String      @id @default(cuid())
  ...
  dependencies    Task[]      @relation("TaskDependencies")
  dependents      Task[]      @relation("TaskDependencies")
}
```

**Логика фильтрации:**
- Задача доступна для выполнения только если `status === 'TODO'`
- И все её зависимости имеют `status === 'DONE'`
- Задачи без зависимостей доступны сразу

### RAG-поиск

Используется библиотека `ai` для векторного поиска:
- **Модель эмбеддингов:** `text-embedding-3-small`
- **Поиск:** Косинусное расстояние через PostgreSQL `vector` тип
- **Инструмент:** `createSearchKnowledgeTool(projectId)`

### Кэширование промптов

Функция `generateTaskPrompt` проверяет поле `generatedPrompt` перед генерацией:
- Если промпт уже существует → возвращается сразу
- Нет лишних вызовов LLM
- Экономия времени и токенов
