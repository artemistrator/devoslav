# Direct Execution Mode - Инструкция

## Быстрый старт

### 1. Требования

- Docker и Docker Compose
- Проект сгенерирован и содержит задачи
- БД запущена и содержит проект

### 2. Запуск приложения

```bash
cd ai-orchestrator
docker compose up -d
```

Приложение доступно на http://localhost:3002

### 3. Использование Direct Execution Mode

1. **Создайте проект** через веб-интерфейс (страница /projects)
2. **Сгенерируйте план** с задачами
3. **Перейдите на страницу плана**: `/project/[projectId]/plan/[planId]`
4. **Переключитесь в режим "Execution"** (вкладка в верхней части страницы)
5. **Нажмите "Start Auto Execution"**
6. **Настройте параметры** в модальном окне:
   - Auto-approve: автоматическое подтверждение команд
   - Cost limit: лимит расходов (опционально)
7. **Нажмите "Start Execution"**

### 4. Как это работает

- AI-агент автоматически выполняет задачи из плана
- Команды отправляются через Sync Client на машину пользователя
- Требуется подтверждение команд (если не включен auto-approve)
- Логи отображаются в реальном времени в Execution Console

### 5. Защита от циклов

Система автоматически определяет повторяющиеся ошибки:
- После 3-х неудачных попыток выполнить одну команду
- Сессия автоматически ставится на паузу
- В логах появляется сообщение: "Execution paused: Failed to fix command..."

### 6. Управление сессией

- **Pause** - приостановить выполнение
- **Resume** - продолжить выполнение
- **Stop** - остановить выполнение полностью

### 7. Troubleshooting

**Ошибка "Project not found":**
- Проверьте, что проект существует в базе данных
- ID проекта можно найти в URL страницы проекта

**Ошибка 404 при старте:**
- Убедитесь, что контейнеры Docker перезапущены после изменений
- Проверьте логи: `docker compose logs app`

**Команды не выполняются:**
- Проверьте, что Sync Client запущен и доступен
- Убедитесь, что порт 3002 открыт

### 8. Мониторинг

- Логи AI-агента: `docker compose logs app -f`
- Логи БД: `docker compose logs db`
- Status сессий: проверьте таблицу `ExecutionSession` в БД
