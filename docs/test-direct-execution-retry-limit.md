# Тест-кейс: Direct Execution — защита от циклов (Retry Limit)

## Цель

Проверить, что при повторяющейся ошибке выполнения сессия **автоматически ставится на паузу** после 3-й попытки с тем же типом ошибки, и в логах/UI отображается сообщение о причине паузы.

## Предусловия

- Проект собран: `npm run build` — без ошибок.
- Docker-образ собран: `docker compose build` — успешно.
- Приложение и sync client (если нужен) запущены.
- В проекте включён/доступен **Direct Execution Mode** и создана execution session с `executionSessionId`.

## Сценарий

1. **Создать задачу**, которая заведомо приводит к ошибке при выполнении команды, например:
   - Описание: «Установи несуществующий пакет: выполни `npm install non-existent-package-xyz-123` и убедись, что он установлен».
   - Или любая задача, где агент будет многократно вызывать одну и ту же падающую команду (одну и ту же ошибку).

2. **Запустить Direct Execution** для этой задачи (старт сессии с привязкой к задаче и передачей `executionSessionId` в инструменты).

3. **Наблюдать за выполнением:**
   - Агент выполнит команду (например, `npm install non-existent-package-xyz-123`).
   - Команда завершится с ошибкой (npm вернёт ненулевой exit code / stderr).
   - Агент получит ошибку в инструменте `executeCommand`, будет вызван `handleCommandError` → инкремент счётчика повторов для данной подписи ошибки.
   - Агент может повторить попытку (до 3 раз с той же подписью ошибки).
   - После **3-й попытки** с той же ошибкой `checkRetryLimit` вернёт `shouldPause: true` и `reason` вида:  
     `Failed to fix the issue after 3 attempts (error signature: ...). Please help.`
   - Сессия переводится в паузу: `pauseSession(executionSessionId)`.

## Ожидаемый результат

- Сессия **автоматически ставится на паузу** после 3-й попытки с той же ошибкой.
- В логах/событиях сессии появляется сообщение вида:
  - **Execution paused: Failed to fix the issue after 3 attempts (error signature: …). Please help.**
- В UI статус сессии — **PAUSED** (или эквивалент).
- Бесконечный цикл «ошибка → повтор → та же ошибка» не продолжается после 3 попыток.

## Что проверяем

| Проверка | Ожидание |
|----------|----------|
| Сборка | `npm run build` — без ошибок компиляции |
| Docker | `docker compose build` — образы собираются |
| Повтор одной и той же ошибки | Счётчик ретраев увеличивается по подписи ошибки (`createErrorSignature`) |
| Лимит 3 попытки | После 3-й одинаковой ошибки `checkRetryLimit` возвращает `shouldPause: true` |
| Пауза сессии | Вызывается `pauseSession`, статус сессии — PAUSED |
| Сообщение пользователю | В логах/UI видно «Execution paused: Failed to fix…» |

## Примечания

- Подпись ошибки считается по паре `command + errorMessage` (хеш), поэтому одна и та же команда с одним и тем же текстом ошибки даёт одну подпись и один счётчик.
- Разные команды или разный текст ошибки дают разные подписи и отдельные счётчики.
- Тест можно повторить с другой заведомо падающей командой (например, `npm run non-existent-script`) для проверки изоляции по подписи.
