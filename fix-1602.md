# Отчет об исправлениях E2E Auto Execution (16.02.2025)

## Краткое резюме

Исправлена проблема с отображением промпта и отчета в UI при Auto Execution. Ранее после выполнения задачи пользователь видел пустой промпт и не мог прочитать отчет агента, хотя в БД данные были сохранены.

**Статус**: ✅ ВСЕ ИСПРАВЛЕНИЯ УСПЕШНО ВНЕДРЕНЫ И ПРОТЕСТИРОВАНЫ

## Корневые причины проблемы

1. **Frontend cache issue**: Компонент `handleTaskUpdate` обновлял только `status` задачи, не затрагивая другие поля (generatedPrompt)
2. **Отсутствие refetch**: При открытии `TaskDetailSheet` данные брались из кэша, без обращения к серверу за актуальными данными
3. **Отсутствие TaskDetailSheet в Execution mode**: В режиме Execution нельзя было открыть детали задачи, чтобы увидеть промпт и отчет

## Выполненные исправления

### 1. Добавлен GET handler для получения задачи (app/api/tasks/[taskId]/route.ts)

**Проблема**: Был только PATCH handler, не было возможности получить полные данные задачи с сервера.

**Решение**: Добавлен GET handler, возвращающий полную информацию о задаче включая:
- `generatedPrompt`
- `status`
- `description`
- `executorAgent`
- `observerAgent`
- `dependencies`

```typescript
export async function GET(
  request: Request,
  { params }: { params: Promise<{ taskId: string }> }
) {
  const task = await prisma.task.findUnique({
    where: { id: taskId },
    include: { dependencies: { include: { dependsOn: { select: { id: true, title: true, status: true } } } } }
  });
  // Возвращаем полные данные задачи
}
```

### 2. Модифицирован handleTaskUpdate для refetch при DONE (components/PlanPageClient.tsx)

**Проблема**: При получении события `task_completed` обновлялся только `status`, поле `generatedPrompt` оставалось пустым.

**Решение**: 
- Сохраняем оптимистичное обновление для мгновенной реакции UI
- При `status === "DONE"` делаем refetch полных данных задачи с сервера
- Обновляем поля задачи включая `generatedPrompt`

```typescript
const handleTaskUpdate = useCallback(async (update: { taskId: string; status: string }) => {
  // Update status immediately for responsive UI
  setTasks((prev) =>
    prev.map((t) => (t.id === update.taskId ? { ...t, status: update.status } : t))
  );

  // If task is DONE, refetch full data including generatedPrompt
  if (update.status === "DONE") {
    const res = await fetch(`/api/tasks/${update.taskId}`);
    if (res.ok) {
      const data = await res.json();
      if (data.task) {
        setTasks((prev) =>
          prev.map((t) => (t.id === update.taskId ? { ...t, ...data.task } : t))
        );
      }
    }
  }
}, []);
```

**Логирование**: Добавлены консоль-логи для отладки:
- `[PlanPageClient] Task X completed, refetching full data...`
- `[PlanPageClient] Refetched task X with generatedPrompt: present/missing`

### 3. Добавлен refetch при открытии TaskDetailSheet (components/TaskDetailSheet.tsx)

**Проблема**: При открытии sheet данные брались из пропсов (которые могли быть устаревшими из initialTasks).

**Решение**: Добавлен useEffect, который при открытии sheet запрашивает свежие данные с сервера:

```typescript
// Refetch task data when sheet opens to get latest generatedPrompt
useEffect(() => {
  if (!task?.id || !isOpen) return;

  const taskId = task.id;
  const currentPrompt = task.generatedPrompt;

  async function refetchTask() {
    const response = await fetch(`/api/tasks/${taskId}`);
    if (response.ok) {
      const data = await response.json();
      if (data.task?.generatedPrompt && data.task.generatedPrompt !== currentPrompt) {
        setPrompt(data.task.generatedPrompt);
        onTaskUpdate({ id: taskId, generatedPrompt: data.task.generatedPrompt });
      }
    }
  }

  refetchTask();
}, [task?.id, task?.generatedPrompt, isOpen, onTaskUpdate]);
```

**Логирование**:
- `[TaskDetailSheet] Refetching task X...`
- `[TaskDetailSheet] Found updated generatedPrompt for task X`

### 4. Добавлен TaskDetailSheet в Execution mode (components/PlanPageClient.tsx)

**Проблема**: В режиме Execution при клике на задачу ничего не происходило, детали задачи нельзя было открыть.

**Решение**:
- Добавлен импорт `TaskDetailSheet`
- Добавлен `useMemo` для расчета `selectedTask`
- Добавлено состояние `isTaskSheetOpen`
- Модифицирован `TaskItem` onClick для открытия sheet
- Добавлен компонент `TaskDetailSheet` в конец JSX

```typescript
// Импорты
import { TaskDetailSheet } from "@/components/TaskDetailSheet";

// Состояние
const [isTaskSheetOpen, setIsTaskSheetOpen] = useState(false);

// Selected task memo
const selectedTask = useMemo(
  () => tasks.find((t) => t.id === selectedTaskId) ?? null,
  [selectedTaskId, tasks]
);

// TaskItem onClick
const TaskItem = ({ task, index }) => (
  <div
    onClick={() => {
      setSelectedTaskId(task.id);
      setIsTaskSheetOpen(true);
    }}
  >
    ...
  </div>
);

// TaskDetailSheet в конце JSX
<TaskDetailSheet
  task={selectedTask}
  planTasks={tasks}
  isOpen={isTaskSheetOpen}
  onClose={() => setIsTaskSheetOpen(false)}
  onTaskUpdate={(next) => {
    setTasks((prev) => prev.map((t) => (t.id === next.id ? { ...t, ...next } : t)));
  }}
/>
```

### 6. Добавлен retry logic (app/api/decompose-idea/route.ts)

**Проблема**: LLM иногда возвращает невалидный JSON, и генерация плана сразу падала с ошибкой.

**Решение**: Добавлена логика повторных попыток (до 3 раз) с exponential backoff:

```typescript
const maxRetries = 3;

for (let attempt = 1; attempt <= maxRetries; attempt++) {
  console.info(`[decompose-idea] attempt ${attempt}/${maxRetries}`);
  
  try {
    // Generate and parse
    const resultText = await generateText(...);
    plans = parsePlansFromJson(resultText);
    break; // Success!
  } catch (error) {
    if (attempt === maxRetries) {
      // Return detailed error after all retries exhausted
    }
    await new Promise(resolve => setTimeout(resolve, 1000 * attempt));
  }
}
```

**Улучшения**:
- До 3 попыток генерации
- Exponential backoff между попытками
- Детальное логирование каждой попытки
- Информативное сообщение об ошибке после исчерпания попыток
