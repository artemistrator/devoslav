### 1. `.cursorrules` - Обновлён до версии 2.0

#### 🔄 Изменён endpoint для получения задач

**Было:**
```bash
curl -s "http://127.0.0.1:3002/api/ai-agent/v1/actions" \
  -H "Content-Type: application/json" \
  -d '{"action":"get_tasks","projectId":"<PROJECT_ID>"}'
```

**Стало:**
```bash
curl -s "http://127.0.0.1:3002/api/ide?action=get_next_task&projectId=<PROJECT_ID>"
```

**Влияние:**
- ✅ Агент получает только 1 задачу вместо всего списка
- ✅ Меньше токенов на запрос
- ✅ Явный фокус на текущей задаче

---

#### 📦 Батч-режим (Batch Mode)

**Новый алгоритм автопилота:**

1. Установить счётчик `tasksInBatch = 0`
2. Получить следующую задачу
3. Выполнить задачу с QA проверкой
4. Увеличить `tasksInBatch++`
5. **Если `tasksInBatch >= 5`:**
   ```
   🛑 БАТЧ ИЗ 5 ЗАДАЧ ВЫПОЛНЕН.

   ✅ Контекст чата переполнен - время сброса.

   📝 ПОЖАЛУСТА, НАЧНИТЕ НОВЫЙ ЧАТ и снова запустите `@orchestrator autopilot`.

   Это защитит от галлюцинаций и гарантирует качество работы.
   ```
6. **Если `tasksInBatch < 5`:** перейти к следующей задаче

**Влияние:**
- ✅ Контекст не переполняется (максимум 5 задач на чат)
- ✅ Чёткие точки "чекпоинтов" для возобновления работы
- ✅ Прагматичный подход: качество > бесшовность

#### 📌 Контекстные якоря (Context Anchors)

**Обязательный формат отчёта:**

```markdown
📌 CURRENT TASK: [ID] [Title]

✅ ВЫПОЛНЕНО:
- [детали что сделано]

📄 ФАЙЛЫ:
- [список созданных/измененных файлов]

🧪 ТЕСТЫ (если требовались):
[логи тестов]

🔧 СБОРКА (если требовалась):
[результаты сборки]

📝 ДОКАЗАТЕЛЬСТВА:
[вывод команд, скриншоты]
```

**Влияние:**
- ✅ Агент всегда знает, какую задачу выполняет
- ✅ QA агент проверяет конкретные артефакты
- ✅ Легче отладить при проблемах

## ⚠️ Важные замечания

### Прагматический подход
- Мы пожертвовали "бесшовностью" ради надёжности
- После 5 задач нужно начать новый чат
- Это предотвращает галлюцинации и гарантирует качество

### QA агент
- Строго проверяет отчёты
- REJECT без доказательств
- Требует конкретные артефакты (файлы, логи, вывод команд)

### Batch Limit (5 задач)
- Оптимизировано для 30-50 задач на проект
- 6-10 батчей для проекта из 30-50 задач
- Чёткие точки для возобновления работы
